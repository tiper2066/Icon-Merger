# ICON Merger 개발 구현 계획

## 1. 프로젝트 목표

ICON Merger는 사용자가 업로드하거나 제공된 SVG 리소스 중에서 메인 아이콘과 병합용 아이콘 또는 병합용 텍스트를 각각 선택한 뒤, 메인 아이콘의 절단 영역에 병합용 리소스를 이어 붙여 하나의 새 아이콘으로 만들고 색상, 선 두께, 크기, 다운로드 포맷을 조정해 내보낼 수 있는 웹 서비스이다.

초기 버전은 `docs/project-overview.md`의 기능 요구사항과 `docs/ui-image.jpg`, `docs/icon-list.png`, `docs/merge-process.png`의 화면 구성을 기준으로 구현한다. 좌측에는 메인 아이콘 세로 목록을, 중앙에는 병합용 리소스 영역을, 우측에는 병합 결과 미리보기와 속성 설정 패널을 배치한다.

## 2. 권장 기술 스택

### Frontend

- Next.js App Router
- React
- TypeScript
- Tailwind CSS
- Shadcn UI
- Lucide React 또는 자체 SVG 아이콘 렌더링 컴포넌트
- React Hook Form, Zod
- TanStack Query 또는 SWR

### Backend

- Next.js Route Handlers 또는 Server Actions
- NextAuth.js/Auth.js 기반 Google OAuth
- Prisma ORM
- PostgreSQL 계열 DB
  - 로컬 개발: Docker Compose의 PostgreSQL
  - 운영 환경: Neon, Supabase Postgres, Railway Postgres, RDS PostgreSQL 중 선택

### Storage

- 초기 개발 및 운영: PostgreSQL DB에 SVG 원본 문자열 저장
- Supabase 단계: Supabase Postgres만 사용하고 Supabase Storage는 사용하지 않음
- 사내 개발망 이전 단계: Docker 기반 PostgreSQL에 동일한 SVG 데이터를 마이그레이션
- 확장 단계: 대용량 파일, PNG/JPG 원본, ZIP 결과물 저장이 필요할 때 MinIO 도입

사용될 아이콘 수는 메인용과 병합용을 합쳐 최대 100개 내외로 예상한다. SVG는 텍스트 기반이고 일반적인 라인 아이콘은 개당 수 KB에서 수십 KB 수준이므로, 초기 운영 범위에서는 별도 Storage 없이 DB 저장만으로 충분하다.

예상 DB 사용량은 보통 100개 기준 수 MB 이내이며, 복잡한 SVG를 넉넉하게 잡아도 수십 MB 수준에 머물 가능성이 높다. 따라서 `Icon.svgContent`를 PostgreSQL `text` 컬럼에 저장하고, `viewBox`, `width`, `height`, 병합 기준점 같은 메타데이터를 함께 저장하는 구조를 기본으로 한다.

향후 사내 개발망으로 이전할 때는 Supabase Postgres 데이터를 Docker 기반 PostgreSQL로 이전한다. MinIO는 당장 필수 구성요소가 아니며, SVG 외 원본 이미지 파일, 다운로드 결과물, ZIP 파일, 대량 에셋 관리가 필요해지는 시점에 도입한다.

### Runtime & Deploy

- Node.js LTS
- Vercel 배포 권장
- Prisma Migration 기반 DB 스키마 관리
- 이미지 변환이 서버 사이드에서 필요할 경우 `sharp` 사용 검토

## 3. 핵심 사용자 흐름

1. 사용자는 Google 계정으로 로그인한다.
2. 서버는 로그인한 Google 이메일이 허용 목록에 포함되어 있는지 확인한다.
3. 허용된 사용자만 앱 메인 화면에 접근할 수 있다.
4. 사용자는 좌측 세로 목록에서 메인 아이콘을 선택한다.
5. 사용자는 중앙의 병합용 리소스 영역에서 병합용 아이콘 또는 병합용 텍스트를 선택한다.
6. 메인 아이콘과 병합용 리소스가 각각 선택되면 우측 패널에서 병합 결과를 미리본다.
7. 사용자는 색상, 선 두께, 크기, 다운로드 포맷을 조정한다.
8. 사용자는 최종 병합 아이콘을 PNG, JPG, SVG 중 원하는 포맷으로 다운로드한다.
9. 관리자는 필요하면 아이콘 업로드 다이얼로그를 열어 SVG 리소스를 추가한다.
10. 관리자는 선택된 리소스를 선택 해제, 전체 선택, 전체 해제, 삭제할 수 있다.
11. 일반 사용자는 공용 아이콘 라이브러리에서 메인 아이콘과 병합용 리소스를 하나씩 선택해 병합/다운로드만 수행한다.

## 4. 화면 구성 계획

### 전체 레이아웃

- 전체 화면은 3개 영역으로 구성한다.
- 좌측 메인 아이콘 영역: 전체 너비의 약 20~25%
- 중앙 병합용 리소스 영역: 전체 너비의 약 50~55%
- 우측 속성 패널: 전체 너비의 약 25%
- 최소 데스크톱 기준으로 우선 구현하고, 이후 태블릿 이하에서는 속성 패널을 하단 또는 Drawer로 전환한다.

### 좌측 메인 아이콘 영역

메인 아이콘은 수량이 적고 앞으로 추가되어도 많지 않을 것으로 예상하므로 좌측에 수직 목록 형태로 배치한다.

- 섹션 제목
- 관리자 기본 상태에서는 아이콘 추가 버튼만 노출
- 관리자 섹션 헤더의 더보기 메뉴에서 전체 선택 제공
- 관리자에게만 선택된 리소스가 있을 때 선택 개수, 선택 해제, 삭제 액션 노출
- 일반 사용자에게는 공용 아이콘 선택 UX만 노출
- 1열 또는 좁은 2열 세로 목록

메인 아이콘 카드는 실제 업로드된 원본 SVG를 기준으로 표시한다. 선택된 아이콘은 파란색 계열 테두리와 옅은 배경으로 강조한다.

### 중앙 병합용 리소스 영역

중앙 영역에는 메인 아이콘에 붙일 수 있는 병합용 리소스를 보여준다. 병합용 리소스는 `아이콘`과 `병합용 텍스트`로 구분한다.

- 병합용 아이콘 섹션
  - 관리자 기본 상태에서는 아이콘 추가 버튼만 노출
  - 관리자 더보기 메뉴에서 전체 선택 제공
  - 관리자에게만 선택된 리소스가 있을 때 선택 개수, 선택 해제, 삭제 액션 노출
  - 일반 사용자는 병합용 텍스트와 상호 배타적으로 1개만 선택
  - 아이콘 그리드

- 병합용 텍스트 섹션
  - 관리자 기본 상태에서는 병합용 텍스트 추가 버튼만 노출
  - 관리자 더보기 메뉴에서 전체 선택 제공
  - 관리자에게만 선택된 리소스가 있을 때 선택 개수, 선택 해제, 삭제 액션 노출
  - 일반 사용자는 병합용 아이콘과 상호 배타적으로 1개만 선택
  - 병합용 텍스트 그리드

병합용 아이콘과 병합용 텍스트는 관리 기능에서는 동일한 선택/삭제 UX를 사용하되, 화면에서는 섹션을 분리해 사용자가 리소스 타입을 명확히 구분할 수 있도록 한다. 병합 결과에는 하나의 병합용 리소스만 사용하므로 한쪽을 선택하면 다른 쪽 선택은 해제된다.

섹션별 관리 액션은 화면 복잡도를 줄이기 위해 공통 패턴을 사용한다. 관리자에게는 선택된 항목이 없을 때 `추가` 버튼과 더보기 메뉴만 보여주고, 선택된 항목이 생기면 `N개 선택됨`, `선택 해제`, `삭제`를 표시한다. `전체 선택`은 항상 노출하지 않고 더보기 메뉴 안에 배치한다. 일반 사용자에게는 관리 액션을 숨긴다.

### 우측 속성 패널

우측 패널은 다음 순서로 구성한다.

1. 패널 제목: `아이콘 속성`
2. 설명 문구: 색상, 두께, 크기를 조정할 수 있다는 안내
3. 초기화 버튼
4. 병합 미리보기 영역
   - 메인 아이콘
   - `+` 표시
   - 병합용 아이콘 또는 병합용 텍스트
   - `=` 표시
   - 최종 병합 아이콘
5. 색상 선택 팔레트
6. 선 두께 슬라이더
7. 크기 슬라이더
8. 다운로드 포맷 선택
9. 다운로드 버튼

### 업로드 다이얼로그

이미지 하단의 다이얼로그 UI를 기준으로 구현한다.

- 제목: `아이콘 업로드`
- 설명: SVG 파일만 업로드 가능하다는 안내
- 드래그 앤 드롭 영역
- 클릭하여 파일 선택
- 선택된 파일 목록 또는 선택 파일명 표시
- 취소 버튼
- 업로드 버튼
- 닫기 버튼

업로드 대상은 다이얼로그를 연 위치에 따라 `MAIN`, `MERGE_ICON`, `MERGE_TEXT` 타입으로 구분한다.

## 5. 기능 요구사항 상세

### 아이콘 목록

- 메인 아이콘, 병합용 아이콘, 병합용 텍스트 목록은 각각 별도로 관리한다.
- 각 리소스는 SVG 원본, 이름, 타입, 소유자, 생성일을 가진다.
- 메인 아이콘은 좌측 세로 목록에 배치한다.
- 병합용 아이콘과 병합용 텍스트는 중앙 영역에서 섹션을 나누어 표시한다.
- 일반 사용자는 각 타입에서 개별 선택만 수행하며, 메인 아이콘 1개와 병합용 아이콘 또는 병합용 텍스트 1개를 병합 대상으로 선택한다.
- 관리자는 각 목록에서 개별 선택, 더보기 메뉴 기반 전체 선택, 선택 해제, 삭제를 수행할 수 있다.
- 관리자 화면의 각 목록 기본 헤더에는 추가 버튼과 더보기 메뉴만 표시하고, 선택 액션은 항목이 선택된 경우에만 표시한다.
- 전체 선택은 더보기 메뉴에 배치해 반복 버튼 노출을 줄인다.
- 삭제는 관리자가 선택한 리소스를 대상으로 하며, 삭제 전 확인 다이얼로그를 표시한다.
- 초기 샘플 아이콘은 제공하지 않으며, 관리자가 필요한 리소스를 직접 업로드한다.

### SVG 업로드

- 허용 포맷은 SVG만 지원한다.
- 파일 확장자와 MIME 타입을 모두 검증한다.
- 서버에서 SVG 내용을 파싱하고 위험한 태그와 속성을 제거한다.
- 허용하지 않을 항목 예시:
  - `script`
  - `foreignObject`
  - 이벤트 핸들러 속성
  - 외부 URL 참조
  - 인라인 JavaScript
- 업로드 파일 크기는 MVP 기준 256KB 이하로 제한한다.
- 업로드 정책은 리소스 타입별로 다르게 적용한다.
  - 메인 아이콘: 한 번에 1개만 업로드
  - 병합용 아이콘: 한 번에 여러 개 업로드 가능
  - 병합용 텍스트: 한 번에 여러 개 업로드 가능
- 메인 아이콘 업로드 시에는 `anchorX`, `anchorY` 좌표 입력을 필수로 한다.
- 병합용 아이콘과 병합용 텍스트는 기준점 좌표를 입력하지 않는다.

### 아이콘 병합

MVP에서는 클라이언트에서 SVG를 합성해 미리보기를 빠르게 제공하고, 다운로드 시 동일한 합성 로직을 사용한다.

권장 병합 방식:

- 메인 SVG와 병합용 리소스 SVG를 각각 정규화한다.
- 메인 아이콘은 병합용 리소스가 붙을 절단 영역의 좌상단 좌표를 `anchorX`, `anchorY`로 가진다.
- 병합용 아이콘 또는 병합용 텍스트의 좌측 상단을 메인 아이콘의 `anchorX`, `anchorY`에 정확히 맞춘다.
- 최종 SVG는 최초 메인 아이콘의 너비와 높이를 유지하지 않고, 병합 결과 전체 bounding box를 기준으로 `viewBox`, `width`, `height`를 다시 계산한다.
- 미리보기는 `메인 + 병합용 = 결과` 구조를 유지하되, 결과 아이콘은 `docs/merge-process.png`의 마지막 줄처럼 절단 영역에 자연스럽게 이어 붙은 형태로 보여준다.

anchor 기반 확장 병합 규칙:

- 메인 아이콘은 실제 업로드 원본과 병합 설명용 경계 데이터를 구분해서 관리한다.
- 경계 데이터는 병합용 리소스가 붙어야 하는 절단 영역을 계산하기 위한 메타데이터이며, 최종 결과물에는 불필요한 경계선이 포함되지 않는다.
- 관리자는 메인 아이콘 업로드 시 병합 기준점 좌표 `anchorX`, `anchorY`를 직접 입력한다.
- 입력한 `anchorX`, `anchorY`는 SVG viewBox 좌표계 기준으로 저장한다.
- 메인 아이콘, 병합용 아이콘, 병합용 텍스트는 각각 높이가 일정하다는 전제를 둔다.
- 병합용 리소스는 메인 아이콘 절단 영역의 상단과 좌측에 맞춰 배치한다.
- 최종 너비는 `max(mainWidth, anchorX + mergeWidth)`로 계산한다.
- 최종 높이는 `max(mainHeight, anchorY + mergeHeight)`로 계산한다.
- 최종 `viewBox`는 `0 0 resultWidth resultHeight`로 설정한다.
- 최종 SVG 루트 안에는 메인 아이콘 그룹과 병합용 리소스 그룹을 각각 배치한다.

병합 메타데이터 예시:

```ts
type MainIconMeta = {
  viewBox: string;
  baseWidth: number;
  baseHeight: number;
  anchorX: number;
  anchorY: number;
};

type MergeResourceMeta = {
  viewBox: string;
  width: number;
  height: number;
  type: "MERGE_ICON" | "MERGE_TEXT";
};
```

병합 좌표 계산 예시:

```text
mergeX = mainIcon.anchorX
mergeY = mainIcon.anchorY
resultWidth = max(mainIcon.baseWidth, mergeX + mergeResource.width)
resultHeight = max(mainIcon.baseHeight, mergeY + mergeResource.height)
```

중첩형 병합, 배지형 병합, 병합 위치 커스터마이징은 MVP 기본 방식이 아니며, 필요 시 향후 확장 옵션으로 검토한다.

### 메인 아이콘 기준점 입력

메인 아이콘은 수량이 적고 병합 기준점의 정확도가 중요하므로, MVP에서는 메인 아이콘 업로드 시 `anchorX`, `anchorY` 좌표를 필수 입력받는다. 메인 아이콘 업로드는 한 번에 1개만 허용한다.

메인 아이콘 업로드 동작:

- SVG 파일을 1개만 선택할 수 있다.
- SVG 미리보기를 표시한다.
- `anchorX`, `anchorY` 숫자 입력 필드를 제공한다.
- 입력한 좌표는 SVG viewBox 좌표계 기준으로 저장한다.
- 입력한 좌표 위치를 미리보기 위에 점 또는 십자선으로 표시한다.
- 저장 전 임시 병합용 리소스를 붙여 결과를 미리 확인할 수 있게 한다.

병합용 아이콘과 병합용 텍스트 업로드 동작:

- 한 번에 여러 SVG 파일을 선택할 수 있다.
- `anchorX`, `anchorY` 입력은 제공하지 않는다.
- 각 SVG의 `viewBox`, `width`, `height`만 정규화해 저장한다.

완료 기준:

- 메인 아이콘 업로드는 단건으로만 가능하다.
- 메인 아이콘 업로드 시 `anchorX`, `anchorY`가 필수로 저장된다.
- 병합용 아이콘과 병합용 텍스트는 다중 업로드가 가능하다.
- 저장된 `anchorX`, `anchorY`가 이후 병합 미리보기와 다운로드 결과에 동일하게 적용된다.
- 기준점 표시는 최종 병합 SVG에 시각 요소로 포함되지 않는다.

### 속성 조정

- 아이콘은 흰색으로 표현한다.
- 색상 박스 테두리는 `rgba(229, 231, 235, 1)`로 설정한다.

선 두께:

- 최소 `0.5px`
- 최대 `3px`

- 최소 `16px`
- 최대 `256px`
- 간격 `4px`

초기화:

- 색상: 검정색
- 선 두께: `1px`
- 크기: `24px`
- 포맷: `SVG`
- 선택 아이콘은 초기화하지 않고 속성 값만 초기화한다.

### 특정 Google 이메일만 허용

환경 변수로 허용 이메일 목록과 관리자 이메일 목록을 관리한다.

```env
GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=
NEXTAUTH_SECRET=
NEXTAUTH_URL=
ALLOWED_GOOGLE_EMAILS=user@example.com,admin@example.com
ADMIN_GOOGLE_EMAILS=admin@example.com
DATABASE_URL=
```

권한 검증 규칙:

- OAuth 로그인 완료 후 profile email을 확인한다.
- `email_verified`가 true인지 확인한다.
- 이메일이 `ALLOWED_GOOGLE_EMAILS`에 포함되어 있지 않으면 로그인 실패 처리한다.
- 미허용 사용자는 앱 접근 권한이 없다는 안내 페이지로 이동한다.
- 서버 API, Server Action, Route Handler에서도 세션과 이메일 허용 여부를 재검증한다.

권장 구현 위치:

- Auth callback의 `signIn` 콜백에서 1차 차단
- Middleware에서 보호 라우트 접근 제어
- 서버 함수에서 `requireAllowedUser()` 유틸로 2차 검증

### 사용자/관리자 역할 분리 계획

관리자만 아이콘 리소스를 업로드/삭제하고, 일반 사용자는 공용 아이콘 리소스를 선택해 병합/속성 조정/다운로드만 수행하는 구조로 전환한다.

환경 변수 계획:

```env
ALLOWED_GOOGLE_EMAILS=user@example.com,admin@example.com
ADMIN_GOOGLE_EMAILS=admin@example.com
```

역할 규칙:

- `ALLOWED_GOOGLE_EMAILS`는 로그인 가능한 전체 이메일 목록이다.
- `ADMIN_GOOGLE_EMAILS`는 관리자 이메일 목록이며, `ALLOWED_GOOGLE_EMAILS`에 포함되어 있어야 한다.
- 관리자 이메일은 특정 회사 도메인으로 제한하지 않고, 테스트 편의를 위해 Gmail 등 외부 Google 계정도 사용할 수 있다.
- `ADMIN_GOOGLE_EMAILS`에 포함된 사용자는 `admin`, 그 외 허용 이메일 사용자는 `user` 역할로 간주한다.
- 로그인 허용 여부와 관리자 여부는 별도 개념으로 분리한다.

권한 규칙:

- 관리자:
  - 메인 아이콘, 병합용 아이콘, 병합용 텍스트 업로드 가능
  - 선택 삭제와 전체 선택 등 리소스 관리용 다중 선택 UX 사용 가능
  - 공용 아이콘 라이브러리를 관리한다.
- 사용자:
  - 공용 아이콘 라이브러리 조회 가능
  - 메인 아이콘, 병합용 아이콘, 병합용 텍스트를 각각 하나씩만 선택 가능
  - 업로드, 삭제, 전체 선택, 선택 해제 등 리소스 관리 기능은 사용하지 않는다.
  - 병합 미리보기, 색상/선 두께/크기 조정, SVG/PNG/JPG 다운로드 가능

사용자 테스트 방법:

- 실제 운영 도메인 계정을 일반 사용자로 테스트하려면 해당 이메일을 `ALLOWED_GOOGLE_EMAILS`에만 추가하고 `ADMIN_GOOGLE_EMAILS`에서는 제외한다.
- 관리자 테스트 계정은 `ALLOWED_GOOGLE_EMAILS`와 `ADMIN_GOOGLE_EMAILS`에 모두 추가한다.
- 로컬 테스트 편의를 위해 관리자 이메일은 Gmail 같은 외부 Google 계정도 사용할 수 있다.

### 아이콘

  - 요청: 타입, 파일 또는 SVG 문자열, 메인 아이콘의 병합 기준점 메타데이터
  - 처리: 인증 확인, SVG 검증, sanitize, normalize, DB 저장

- `DELETE /api/icons`
  - 선택 리소스 일괄 삭제
  - 요청: 리소스 ID 배열
  - 처리: 관리자 권한 검증 후 삭제

  - 관리자 전용 메인 아이콘 anchor 좌표 수정
  - 요청: `anchorX`, `anchorY`
  - 처리: 관리자 권한과 메인 아이콘 타입 검증 후 DB 좌표 업데이트

### 역할 기반 공용 아이콘 라이브러리 전환 계획

현재 구현은 아이콘을 로그인 사용자 `userId` 기준으로 조회하고 업로드/삭제한다. 사용자/관리자 역할 분리 후에는 아이콘 리소스를 앱 공용 라이브러리로 취급한다.

데이터 정책:

- `Icon`은 업로드한 관리자 추적을 위해 `userId`를 유지할 수 있다.
- 일반 조회 화면과 `GET /api/icons`는 로그인 사용자 본인 소유가 아니라 공용 아이콘 라이브러리 전체를 조회한다.
- 업로드와 삭제 권한은 관리자에게만 허용한다.
- 관리자 여러 명이 있는 경우 각 관리자가 업로드한 리소스를 모두 공용 라이브러리에 포함한다.
- 향후 감사 로그가 필요하면 업로드/삭제한 관리자 이메일과 시각을 별도 기록하는 확장을 검토한다.

API 권한 계획:

- `GET /api/icons`
  - 허용 사용자와 관리자 모두 접근 가능
  - 공용 라이브러리 아이콘을 타입별로 조회
- `POST /api/icons`
  - 관리자만 접근 가능
  - 일반 사용자가 호출하면 `403 Forbidden` 응답
- `DELETE /api/icons`, `DELETE /api/icons/[id]`
  - 관리자만 접근 가능
  - 일반 사용자가 호출하면 `403 Forbidden` 응답
- `PATCH /api/icons/[id]`
  - 관리자만 접근 가능
  - 메인 아이콘의 `anchorX`, `anchorY`만 수정 가능

서버 유틸 계획:

- `getCurrentUser()`는 기존처럼 허용된 로그인 사용자를 보장한다.
- `getCurrentUserWithRole()` 또는 `requireAdminUser()` 같은 역할 유틸을 추가한다.
- Route Handler에서는 UI 권한만 믿지 않고 관리자 권한을 서버에서 재검증한다.

### 병합

## 11. SVG 처리 기술 상세

### SVG Sanitize

보안상 SVG를 이미지가 아니라 잠재적인 HTML/XML 입력으로 취급해야 한다.

처리 단계:

1. 파일 크기 검사
2. XML 파싱 가능 여부 검사
3. 루트가 `svg`인지 확인
4. 허용 태그만 유지
5. 위험 속성 제거
6. 외부 URL 참조 제거
7. 정규화된 SVG 문자열 생성

권장 라이브러리:

- `isomorphic-dompurify`
- `sanitize-html`
- 서버 전용 XML 파서

### SVG Normalize

병합과 색상 제어를 안정적으로 하기 위해 업로드된 SVG를 정규화한다.

- `viewBox`가 없으면 width/height 기반으로 생성한다.
- 메인 아이콘은 `baseWidth`, `baseHeight`, `anchorX`, `anchorY`를 기준으로 병합 좌표계를 갖는다.
- 병합용 아이콘과 병합용 텍스트는 정해진 높이 체계에 맞춰 `width`, `height`, `viewBox`를 정규화한다.
- `stroke`, `fill`은 정책에 맞게 `currentColor`로 치환한다.
- 불필요한 metadata, title, desc는 선택적으로 보존하거나 제거한다.

### SVG Merge

합성 결과는 anchor 기반 확장 SVG 형태를 목표로 한다.

```svg
<svg viewBox="0 0 resultWidth resultHeight" width="resultWidth" height="resultHeight" xmlns="http://www.w3.org/2000/svg">
  <g color="currentColor" stroke="currentColor">
    <g data-layer="main">...</g>
    <g data-layer="merge" transform="translate(anchorX anchorY)">...</g>
  </g>
</svg>
```

`resultWidth`와 `resultHeight`는 메인 아이콘과 병합용 리소스가 차지하는 전체 영역을 기준으로 계산한다. 이 때문에 최종 병합 아이콘의 너비와 높이는 최초 메인 아이콘과 달라질 수 있다.

속성 적용:

- 색상은 CSS `color` 또는 SVG 속성으로 적용
- 선 두께는 `stroke-width`로 적용
- 크기는 결과 SVG의 비율을 유지한 상태에서 다운로드 변환 크기에 적용

## 12. Shadcn UI 및 Penta Design System 활용 계획

UI 디자인 구현은 `docs/penta-design-system.md`와 동일 내용의 `docs/penta-design-system.html`을 기준으로 한다. Shadcn UI는 접근성, 상호작용, 컴포넌트 구조를 빠르게 구성하기 위한 기반으로 사용하고, 시각 스타일은 Penta Design System의 토큰을 Tailwind CSS theme와 Shadcn UI CSS 변수에 매핑해 적용한다.

사용할 컴포넌트:

- `Button`
- `Dialog`
- `Card`
- `Separator`
- `Slider`
- `ToggleGroup`
- `Tooltip`
- `ScrollArea`
- `AlertDialog`
- `DropdownMenu` 또는 `Popover`
- `Skeleton`
- `Toast` 또는 `Sonner`

### Penta Design Token 적용 기준

- 컬러
  - 주요 액션: `Primary/500` `#1E6FFF`
  - 주요 액션 hover: `Primary/600` `#1858CC`
  - 주요 액션 active: `Primary/700` `#124199`
  - 기본 텍스트: `Neutral/900` `#111620`
  - 보조 텍스트: `Neutral/600` `#545D70`
  - 비활성 텍스트: `Neutral/400` `#9AA1B0`
  - 기본 배경: `Neutral/0` `#FFFFFF`
  - 서브 배경: `Neutral/50` `#F7F8FA`
  - 기본 보더: `Neutral/200` `#D9DCE3`
  - 삭제/위험 액션: `Error/500` `#EF4444`

- 타이포그래피
  - 기본 폰트는 Pretendard를 사용한다.
  - 페이지 제목은 Heading 3 또는 Heading 4 기준을 사용한다.
  - 본문과 설명 문구는 Body Small 또는 Body 기준을 사용한다.
  - 버튼과 배지는 Label 또는 Body Small의 Medium/SemiBold 기준을 사용한다.

- 스페이싱
  - 8pt grid를 기본으로 사용한다.
  - 카드 내부 여백은 `spacing/6` `16px` 또는 `spacing/7` `20px`를 우선 사용한다.
  - 섹션 간격은 `spacing/8` `24px` 이상을 기준으로 한다.

- Radius와 Shadow
  - 버튼과 입력창은 `radius/md` `8px`를 사용한다.
  - 아이콘 카드와 패널은 `radius/lg` `12px`를 사용한다.
  - 업로드 다이얼로그와 우측 패널은 필요 시 `radius/xl` `16px`를 사용한다.
  - 카드에는 기본적으로 `shadow/md`, 모달에는 `shadow/lg` 기준을 적용한다.

UI 스타일 방향:

- Penta Design System의 흰색 배경, Neutral 계열 보더, Primary Blue 중심의 액션 컬러를 사용
- 선택 상태는 `Primary/500` 테두리와 `Primary/50` 배경으로 강조
- 주요 액션 버튼은 `Primary/500`, hover는 `Primary/600`, active는 `Primary/700` 사용
- 삭제 버튼은 `Error/500` 사용
- 속성 패널은 우측 고정 컬럼 형태
- 메인 아이콘 영역은 좁은 세로 목록으로 구성
- 병합용 리소스 영역은 넓은 그리드로 구성하고 `아이콘`, `병합용 텍스트` 섹션을 명확히 구분

## 13. 반응형 및 접근성 계획

반응형:

- 데스크톱: `grid-cols-[minmax(180px,0.8fr)_minmax(420px,2fr)_minmax(280px,1fr)]`
- 좌측: 메인 아이콘 세로 목록
- 중앙: 병합용 아이콘과 병합용 텍스트 섹션형 그리드
- 태블릿: 메인 아이콘과 병합용 리소스를 상단 작업 영역에 배치하고 속성 패널은 하단 이동 검토
- 모바일: 메인 아이콘, 병합용 아이콘, 병합용 텍스트를 탭으로 분리하고 속성 패널은 Drawer 형태로 제공

접근성:

- 모든 버튼에 명확한 텍스트 또는 `aria-label` 제공
- 아이콘 카드 선택 상태에 `aria-pressed` 적용
- 업로드 영역은 키보드로 파일 선택 가능해야 함
- 색상 선택은 색상만으로 의미를 전달하지 않고 선택 표시를 함께 제공
- Penta Design System 색상 토큰 사용 시 WCAG AA 대비 함께 확인
- 슬라이더는 현재 값을 텍스트로 표시

## 20. 확정된 주요 의사결정

개발 시작 전에 논의된 주요 항목은 다음과 같이 확정한다.

1. 메인 아이콘 병합 기준점 관리 방식
   - 메인 아이콘 업로드는 한 번에 1개만 허용한다.
   - 메인 아이콘 업로드 시 관리자가 `anchorX`, `anchorY` 좌표를 필수 입력한다.
   - 입력한 좌표는 SVG viewBox 좌표계 기준으로 저장한다.
   - 입력 좌표는 업로드 미리보기 위에 점 또는 십자선으로 표시해 확인할 수 있게 한다.
   - 병합용 아이콘과 병합용 텍스트는 기준점 좌표를 입력하지 않는다.

2. 선 두께 간격
   - `0.5px` 단위로 확정한다.
   - 값 후보는 `0.5`, `1`, `1.5`, `2`, `2.5`, `3`으로 제공한다.

3. 초기 샘플 아이콘 제공 여부
   - 초기 샘플 아이콘은 제공하지 않는다.
   - 관리자가 운영 중 필요한 메인 아이콘, 병합용 아이콘, 병합용 텍스트를 직접 업로드한다.
   - 메인 아이콘은 단건 업로드, 병합용 아이콘과 병합용 텍스트는 다중 업로드로 운영한다.

4. SVG 저장 방식
   - 초기 개발과 운영은 PostgreSQL DB 저장으로 확정
   - MinIO는 대용량 파일, ZIP 결과물, SVG 외 원본 파일 저장이 필요할 때 도입

5. 다중 선택 병합 규칙
   - MVP에서는 대표 조합 1개를 미리보기와 다운로드 대상으로 사용한다.
   - 여러 조합 일괄 생성과 ZIP 다운로드는 향후 확장 기능으로 둔다.

6. 사용자/관리자 역할과 공용 아이콘 라이브러리
   - 로그인 허용 목록과 관리자 목록은 분리한다.
   - `ALLOWED_GOOGLE_EMAILS`는 로그인 가능한 전체 이메일 목록이다.
- `ADMIN_GOOGLE_EMAILS`는 업로드/삭제/anchor 수정 권한이 있는 관리자 이메일 목록이다.
   - 관리자 이메일은 회사 도메인으로 제한하지 않고 Gmail 등 외부 Google 계정도 사용할 수 있다.
   - 아이콘 리소스는 관리자 개인 소유가 아니라 앱 공용 라이브러리로 제공한다.
   - 일반 사용자는 공용 라이브러리에서 메인 아이콘, 병합용 아이콘, 병합용 텍스트를 각각 하나씩만 선택한다.
   - 일반 사용자 화면에서는 업로드, 삭제, 선택 해제, 전체 선택 등 관리 기능을 숨긴다.
   - 서버 API에서는 업로드/삭제/anchor 수정 요청마다 관리자 권한을 재검증한다.

### 11-2단계 구현 결과: 접근성 속성 보강

작업 화면의 주요 인터랙션에 접근성 속성을 보강했다.

- 아이콘 섹션을 제목/설명과 `aria-labelledby`, `aria-describedby`로 연결했다.
- 아이콘 목록에 `role="list"`와 카드 wrapper의 `role="listitem"`을 적용하고, 빈 상태에서는 목록 role을 사용하지 않도록 했다.
- 섹션별 선택 개수, 업로드 선택 파일, 삭제/업로드/anchor/다운로드 오류 메시지를 스크린리더가 인지할 수 있도록 `aria-live` 또는 `role="alert"`를 적용했다.
- 삭제 확인, anchor 수정, 업로드 다이얼로그에 제목/설명 연결과 `aria-modal`, `role="dialog"`를 정리했다.
- 업로드 드롭존에 명확한 접근성 이름과 설명을 부여하고, Enter/Space 키로 파일 선택 창을 열 수 있게 했다.
- 색상 선택 그룹에 접근성 이름을 추가하고, 색상 버튼의 선택 상태를 텍스트와 `aria-pressed`로 함께 전달한다.
- 선 두께와 다운로드 크기 range 입력에 현재 값을 `aria-valuetext`로 제공한다.

### 11-3단계 구현 결과: 반응형 레이아웃 조정

모바일 우선 작업 흐름에 맞춰 작업 화면의 반응형 레이아웃을 조정했다.

- 모바일 헤더는 상단 sticky 구조로 유지하고, 사용자 정보와 로그아웃은 햄버거 메뉴 안으로 이동했다.
- 데스크톱에서는 기존처럼 헤더에 사용자 정보와 로그아웃을 노출한다.
- 모바일 본문은 `메인 아이콘`을 먼저 보여주고, 그 아래에서 병합 리소스를 선택하도록 세로 흐름으로 구성했다.
- 병합용 아이콘과 병합용 텍스트는 모바일에서 탭으로 전환해 좁은 화면에서 한 종류씩 집중해서 선택할 수 있게 했다.
- 태블릿에서는 메인 아이콘과 병합 리소스를 2컬럼으로 배치하고, 속성 패널은 하단 전체폭으로 이동한다.
- 데스크톱에서는 기존 3컬럼 레이아웃을 유지한다.
- 모바일에서는 메인 아이콘과 병합 리소스 선택 전에는 속성 패널을 숨긴다.
- 병합 결과를 만들 수 있는 선택이 완료되면 하단 고정 `결과 조정하기` 플로팅 버튼을 표시한다.
- 플로팅 버튼을 누르면 속성 패널이 우측에서 좌측으로 슬라이드되는 Drawer로 열린다.
- 속성 Drawer가 열려 있거나 선택이 해제되어 병합 결과가 없어지면 플로팅 버튼은 표시하지 않는다.

### 7단계 구현 결과

SVG 정규화 정보를 사용한 anchor 기반 병합 미리보기를 구현했다.

- `src/lib/svg/merge-svg.ts`에 클라이언트와 다운로드 단계에서 재사용 가능한 순수 SVG 병합 유틸을 추가했다.
- 병합 유틸은 각 SVG의 루트 `viewBox`와 내부 콘텐츠를 분리하고, 메인 아이콘의 `anchorX`, `anchorY`에 병합용 리소스의 좌측 상단을 맞춘다.
- 메인 SVG와 병합용 리소스를 각각 `<g data-layer="main">`, `<g data-layer="merge">` 그룹으로 배치한다.
- 결과 SVG의 `viewBox`, `width`, `height`는 `max(mainWidth, anchorX + resourceWidth)`, `max(mainHeight, anchorY + resourceHeight)` 기준으로 재계산한다.
- viewBox 시작점이 0이 아닌 SVG도 결과 좌표계 기준으로 배치되도록 minX/minY를 보정한다.
- MVP 규칙에 따라 선택된 메인 아이콘 1개와 선택된 병합용 리소스 1개를 대표 조합으로 사용한다.
- 대표 리소스는 병합용 아이콘 선택을 우선하고, 없으면 병합용 텍스트 선택을 사용한다.
- 우측 속성 패널 미리보기 영역에서 `메인 + 리소스 = 결과` 구조로 실제 SVG를 표시한다.
- 선택이 부족하거나 메인 아이콘에 anchor 좌표가 없을 때 안내 문구를 표시한다.
- 2개 이상 선택된 경우 마지막으로 직접 선택한 메인/리소스를 대표 미리보기 대상으로 사용한다.
- 우측 속성 패널에 `대표 미리보기` 영역을 추가해 현재 표시 중인 조합 이름을 `메인 + 리소스` 형태로 보여준다.
- 다중 선택 상태에서는 마지막으로 선택한 항목을 대표로 표시한다는 안내 문구를 제공한다.
- 속성 패널 미리보기에서도 라인 아이콘이 채워져 보이지 않도록 SVG preview 전용 fill/stroke 보정 클래스를 추가했다.

### 5단계 구현 결과

SVG 업로드 다이얼로그와 서버 사이드 SVG 검증/저장을 구현했다.

- 작업 화면을 `src/components/icon-workspace.tsx` 클라이언트 컴포넌트로 분리하고 `추가` 버튼에서 업로드 다이얼로그를 열도록 연결했다.
- 메인 아이콘은 단건 SVG 업로드와 `anchorX`, `anchorY` 숫자 입력을 제공한다.
- 메인 아이콘 다이얼로그는 선택한 SVG를 확대 미리보기로 보여주고 입력 좌표 위치를 점/십자선으로 표시한다.
- 메인 아이콘 미리보기 영역을 클릭하거나 드래그하면 `anchorX`, `anchorY` 좌표가 SVG viewBox 좌표계 기준으로 갱신된다.
- 앵커 좌표는 `0.5px` 단위로 반올림한다.
- 병합용 아이콘과 병합용 텍스트는 다중 SVG 선택을 허용한다.
- 업로드 성공 후 `router.refresh()`로 DB 목록을 다시 읽어 화면에 표시한다.
- `POST /api/icons` 업로드 API를 추가했다.
- 서버 유틸 `src/lib/svg/process-svg.ts`에서 SVG 확장자, MIME 타입, 256KB 크기 제한, XML 파싱, 루트 `svg` 검증을 수행한다.
- `sanitize-html`로 위험 태그/속성을 제거하고, `script`, `foreignObject`, 이벤트 핸들러, 외부 URL/JavaScript 참조가 저장되지 않도록 처리한다.
- `viewBox`가 있으면 해당 크기를 기준으로, 없으면 `width`/`height` 기준으로 `viewBox`, `width`, `height`를 정규화한다.
- 메인 아이콘은 `baseWidth`, `baseHeight`, `anchorX`, `anchorY`를 함께 저장한다.
